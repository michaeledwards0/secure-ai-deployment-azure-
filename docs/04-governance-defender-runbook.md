# Phase 4 Runbook – Governance & Defender for Cloud

## Purpose
Configure governance, compliance monitoring, and centralized telemetry.

## Prerequisites
- Phases 1–3 complete
- Azure AI Services deployed
- Appropriate RBAC permissions

## Steps
1. Verify Azure Policy compliance.
2. Enable Defender for AI Services, Servers, and Key Vault.
3. Capture Secure Score.
4. Create Log Analytics Workspace.
5. Configure Diagnostic Settings.
6. Enable Azure Activity Logs.
7. Enable Microsoft Entra diagnostic logs.
8. Archive diagnostics to Storage Account.
9. Assign Reader role.
10. Validate telemetry ingestion.

## Validation Checklist
- [ ] Policy verified
- [ ] Defender plans enabled
- [ ] Secure Score captured
- [ ] Log Analytics receiving logs
- [ ] Activity logs enabled
- [ ] Entra logs enabled
- [ ] Ready for Phase 5
